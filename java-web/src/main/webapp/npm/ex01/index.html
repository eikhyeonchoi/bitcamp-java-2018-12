외부 라이브러리 사용 - CDN(Content Delivery Network) 방식

    정의 = 외부라이브러리를 사용할 때 원래의 URL을 그대로 링크건다
    [장점]
    - 우리의 서버에 다운로드 할 필요 없어 관리가 편함
    - 웹브라우저가 각 사이트별로 파일을 캐시에 담아두기 때문에
      사용자가 다른 시스템을 방문했을 때, 이미 다운로드한 라이브러리면 
      다시 받지 않기 때문에 페이지 로딩 속도 빨라짐
      (java-web-project에서 한게 CDN 방식임)
      
    [단점]
    - 라이브러리를 가지고있는 서버에 연결되지 않으면 시스템의 기능을 온전히 사용못함
    - 라이브러리가 외부에 있어서 마음대로 통제 안됨
      라이브러리가 있는 서버쪽에서 해킹 코드를 삽입해도 막을 방법없음
